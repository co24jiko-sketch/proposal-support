# 技術提案書サポート — パイロット運用手順

最終更新: 2026-06-20（Phase 1〜4 完了・公開確認済み）

## パイロットの目的

入札図書から技術提案書の初稿作成・適合チェック・承認・PDF 出力までを、**実際の業務に近い流れ**で試し、使い勝手と不足機能を洗い出す。

| 用途 | URL |
|---|---|

| 案件一覧 | https://proposal-support.vercel.app/proposal |

※ 未ログインで `/proposal` を開くと、自動的にログイン画面へ移動します。

## アカウント（3 ロール）

| ロール | 用途 | メール | パスワード |
|---|---|---|---|
| **担当者** | 案件作成・編集・承認申請・PDF 出力 | Supabase で個別作成（下記参照） | 作成時に設定 |
| **部長** | 1 次承認 | `manager@pilot.local` | `PilotManager2026` |
| **支社長** | 2 次承認 | `director@pilot.local` | `PilotDirector2026` |

### 担当者アカウントについて

- 担当者は **1 人 1 アカウント**（Supabase Dashboard → Authentication → Users で作成）
- 新規ユーザーは自動的に `assignee` ロールになる（`add_auth_profiles.sql` 実行済み前提）
- **自分が作成した案件だけ** 編集・アップロードできる（ヘッダーの表示名＝案件の担当者名）

### ログインできないとき

| 症状 | 対処 |
|---|---|
| 支社長ログイン失敗 | Dashboard でユーザーを削除 → 再作成（Auto Confirm）→ SQL で `role = 'director'` |
| パスワードリセットメール失敗 | rate limit のことがある。Dashboard から再作成を優先 |
| 部長で他人の案件を編集できない | 正常動作。担当者で新規案件を作成する |

## 通し操作手順（担当者）

**所要目安: 15〜20 分**（初回）

### 1. ログイン → 新規案件

1. ログイン画面で担当者アカウントを入力
2. **新規案件** を作成（工事名はわかりやすい名前、例: 「パイロット試行 6/20」）
3. 自動的に **チェックリスト** タブへ移動

### 2. チェックリスト

1. （任意）**PDFをアップロード** — 入札図書 PDF（**4MB 以下**）
2. **採点基準マスタ** で「地質調査（標準・仮）」等を選び **適用する**
3. **確定して初稿生成へ**

※ 採点項目 0 件でも確定可能（適合チェックはスキップ扱い）

### 3. 文案・Word

1. **初稿を一括生成**
2. **再取込して適合チェックへ**（または適合チェックタブへ移動）

### 4. 適合チェック

1. ○△× を確認（地質調査（標準・仮）の想定: 調査方針・方法=○、品質・安全=×）
2. △× がある場合は理由を入力
3. **承認を申請する**

### 5. 承認（部長 → 支社長）

1. **ログアウト** → 部長アカウントでログイン
2. 案件一覧から対象案件を開く → **承認** タブ → **承認**
3. **ログアウト** → 支社長アカウントでログイン → 同様に **承認**

### 6. PDF 出力（担当者）

1. 担当者で再ログイン
2. **承認** タブ → **提出版 PDF を出力** → ダウンロード

### 7. 履歴タブ（Phase 4）

**履歴** タブで以下を確認:

- **版履歴**: v1（初稿）、v2（適合チェック）など
- **操作ログ**: 案件作成、採点基準適用、承認申請、承認 など

※ 公開サイト（Vercel）の時刻は UTC 表示。日本時間より約 9 時間早く見える（データは正しく保存されている）

## よくあるトラブル

| 症状 | 原因 | 対処 |
|---|---|---|
| PDF アップロード失敗 | 4MB 超 | 4MB 以下に圧縮 |
| PDF アップロード失敗 | 他人の案件を開いている | 自分で新規案件を作成 |
| 「この案件を操作する権限がありません」 | 担当者不一致 | 上記と同じ |
| 履歴が空 | SQL 未実行 / 古いデプロイ | `add_case_history.sql` 実行済みか、Vercel Ready 確認 |
| ログイン画面が開かない（ローカル） | dev サーバー停止・固まり | `taskkill /F /IM node.exe` → `npm run dev` |

## パイロットで試さないこと（未実装・制限）

- 新規案件ウィザード Step 2 からの PDF アップロード（チェックリストタブから行う）
- Word を手元で編集して再アップロード
- 4MB を超える入札図書 PDF のアップロード（Vercel 制限）
- 採点基準マスタの画面からの編集（コード内の仮マスタ 2 件のみ）
